# 배포 전 미리보기 가이드

배포하기 전에 프로덕션 빌드를 로컬에서 미리 확인하는 방법입니다.

## 빠른 미리보기

### 방법 1: 개발 서버 (개발 중 확인)

개발 중에 실시간으로 변경사항을 확인:

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:5173` 접속

### 방법 2: 프로덕션 빌드 미리보기 (배포 전 확인) ⭐

배포될 실제 버전과 동일하게 확인:

```bash
# 1. 의존성 설치 (처음 한 번만)
npm install

# 2. 프로덕션 빌드 생성
npm run build

# 3. 빌드된 결과물 미리보기
npm run preview
```

브라우저에서 `http://localhost:4173` (또는 표시된 포트) 접속

## 두 방법의 차이점

| 구분 | `npm run dev` | `npm run build` + `npm run preview` |
|------|---------------|--------------------------------------|
| 용도 | 개발 중 실시간 확인 | 배포 전 최종 확인 |
| 빌드 | 개발 모드 (느림) | 프로덕션 모드 (최적화됨) |
| 속도 | 빠름 (HMR 지원) | 느림 (최적화된 빌드) |
| 파일 크기 | 큰 파일 (디버깅 정보 포함) | 작은 파일 (압축됨) |
| 실제 배포와 동일 | ❌ | ✅ |

## 권장 워크플로우

1. **개발 중**: `npm run dev` 사용
   - 코드 작성 및 수정
   - 실시간으로 변경사항 확인

2. **배포 전**: `npm run build` + `npm run preview` 사용
   - 프로덕션 빌드 생성
   - 실제 배포될 버전 확인
   - 문제 발견 시 수정 후 다시 빌드

3. **배포**: GitHub에 push → Vercel 자동 배포

## 문제 해결

### 빌드 에러가 발생하면:
```bash
# node_modules 삭제 후 재설치
rm -rf node_modules
npm install
npm run build
```

### 미리보기 포트가 이미 사용 중이면:
- Vite가 자동으로 다른 포트를 사용합니다
- 터미널에 표시된 포트 번호를 확인하세요

### 변경사항이 반영되지 않으면:
- `npm run build`를 다시 실행해야 합니다
- `preview`는 빌드된 파일을 보여주므로, 코드 변경 후 다시 빌드 필요
